# V1 功能列表 / Roadmap：AI Chat 多 Agent 分析工作台

- **文档版本**：V1.0
- **日期**：2026-03-25
- **适用范围**：MVP → V1

---

# 1. 文档目标

本文档用于定义 AI Chat 多 Agent 分析工作台从 MVP 到 V1 的功能范围、优先级、分阶段目标和推进顺序，便于产品、研发与设计协同排期。

---

# 2. 版本策略

## 2.1 核心策略
采用“先闭环、再增强、后扩展”的版本推进方式：

- **MVP**：先把最小双 Agent 闭环跑通
- **V1**：在 MVP 基础上增强体验、稳定性和可用性
- **VNext**：再考虑模板化、更多 Agent、更多数据源

## 2.2 版本原则
- 优先做用户最直接感知的价值
- 优先做高频核心链路
- 优先做能验证产品假设的能力
- 延后复杂但非核心的系统建设

---

# 3. MVP 范围

## 3.1 MVP 目标
完成“上传 → 问答 → 报表 → 继续追问”的最小可用闭环。

## 3.2 MVP 必做功能

### A. Session 工作区
- 创建 Session
- 查看 Session 列表
- 打开 Session
- Session 按更新时间排序

### B. 通用对话 Agent
- 输入问题
- 获取回复
- 结合当前 Session 上下文回答
- 区分 conversation agent 身份

### C. 报表 Agent
- 手动触发生成报告
- 无上传时禁止生成
- 生成通用分析报告
- 区分 report agent 身份

### D. 文件上传
- 支持 CSV / Excel / Text / Markdown
- 文件类型校验
- 文件大小限制
- 上传数量限制
- 生成摘要和结构预览

### E. 报表展示
- 展示报告列表
- 查看最新报告摘要
- 记录报告与上传内容关联

### F. 持久化
- 保存 Session
- 保存消息
- 保存上传文件记录
- 保存报告记录

---

# 4. MVP 优先级

## P0（必须完成）
- Session 工作区可用
- 上传可用

- 本地持久化可用

# 5. V1 范围

V1 在 MVP 基础上，不改变产品主结构，但提升易用性、稳定性和感知价值。

## 5.1 V1 目标
- 让双 Agent 体验更清晰
- 让上传数据与分析结果的关系更透明
- 让报表能力更稳定、更可信
- 让产品具备更强的演示和试用价值

## 5.2 V1 建议功能

### A. 双 Agent 体验增强
- 在 UI 中更明确区分两个 Agent
- 对话 Agent 提示何时适合生成报告
- 报表生成后突出显示来源与结果

### B. 上传体验增强
- 更好的上传空状态引导
- 更好的字段/摘要可视化
- 更清晰的格式与限制提示

### C. 报表体验增强
- 固化通用分析报告结构
- 优化报告摘要质量
- 提高报表可读性和追问承接能力

### D. 稳定性增强
- 更完整的错误提示
- 更完善的边界情况处理
- 更稳定的 Session 恢复体验

### E. 可演示能力增强
- 更友好的默认文案
- 更清晰的成功/失败反馈
- 更适合演示的默认页面状态

# 6. 分期路线图

# Phase 0：基础闭环验证
## 目标
验证双 Agent 最小链路能跑通。

## 范围
- Session
- Upload
- Chat
- Report
- Local persistence

## 验收标准
- 完整主流程可跑通
- 不依赖外部复杂基础设施

---

# Phase 1：MVP 发布版
## 目标
形成对外可展示的 MVP 产品形态。

## 范围
- 双 Agent 角色明确
- 页面结构稳定
- 上传、聊天、报表体验打磨
- 基础错误处理完成

## 验收标准
- 用户能理解双 Agent
- 用户能顺利生成第一份报告
- 基本异常场景有提示

---

# Phase 2：V1 体验增强
## 目标
提升产品完整度和可信度。

## 范围
- 优化报表结构
- 优化上传结果展示
- 优化对话承接逻辑
- 增强引导和空状态设计

## 验收标准
- 用户路径更顺滑
- 报表更像可交付成果
- 上传与分析关系更透明

# 7. 功能优先级矩阵

## 高价值 + 高优先级
- 上传后可直接提问
- 生成通用分析报告
- 报告可在对话中继续使用
- 双 Agent 身份清晰

## 高价值 + 中优先级
- 对话 Agent 提示生成报告
- 更清晰的报告结构展示
- 更好的上传摘要展示

## 中价值 + 低优先级
- 报表模板切换
- 多文件筛选后生成报表
- 更复杂的 Session 元信息管理

## 低价值 + 低优先级（本期不做）
- 权限系统
- 协作评论
- 企业系统接入
- 自动任务链
- 检索增强

# 10. 结论

路线建议如下：

## 当前先做
- 双 Agent 最小闭环
- 手工上传分析
- 通用分析报告
- 单用户工作区

## V1 再增强
- 体验
- 稳定性
- 引导
- 成果表达

## 更后面再扩展
- 模板
- 更多 Agent
- 企业接入
- 协作与权限

一句话总结：

> 先把“能用、好懂、能演示”的双 Agent 分析闭环做出来，再逐步扩展为更强的 AI 分析平台。
